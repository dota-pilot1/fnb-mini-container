학습 계획 — Tabulator로 관리자 페이지 구현하기
================================================================

■ 목표
  현업에서 사용하는 그리드 라이브러리(Tabulator) 활용법을 익히고,
  관리자 페이지의 핵심 UI 패턴(조회/등록/수정/삭제)을 구현

■ Tabulator를 쓰는 이유
  - 현업 관리자 페이지는 대부분 그리드 중심 UI
  - 페이지네이션, 정렬, 필터, 인라인 수정 등을 직접 구현하면 복잡
  - Tabulator가 이 기능을 대부분 내장으로 제공

----------------------------------------------------------------

1단계 — Tabulator 기본 사용법
----------------------------------------------------------------
  - 설치 및 React 연동
  - 컬럼 정의, 데이터 바인딩
  - 페이지네이션 (클라이언트 / 서버사이드)
  - 정렬, 필터

  [ 학습 포인트 ]
  - Tabulator 공식 문서 읽는 법
  - React에서 ref로 Tabulator 인스턴스 제어


2단계 — 서버사이드 페이징 연동
----------------------------------------------------------------
  - 백엔드 페이징 API 설계
    GET /api/brands?page=1&size=20&sort=name&filter=...
  - Tabulator ajaxURL, ajaxParams 설정
  - 응답 포맷 맞추기 (last_page, data 형식)

  [ 학습 포인트 ]
  - 서버사이드 페이징 vs 클라이언트사이드 차이
  - MyBatis 페이징 쿼리 (LIMIT, OFFSET)


3단계 — 인라인 수정 + 저장
----------------------------------------------------------------
  - 셀 클릭 시 인라인 편집 활성화
  - 수정된 행 감지 → 저장 버튼 클릭 시 API 호출
  - 행 추가 / 행 삭제

  [ 학습 포인트 ]
  - Tabulator cellEdited 이벤트
  - 변경된 데이터만 추려서 PUT/PATCH 요청


4단계 — 관리자 페이지 실제 구현
----------------------------------------------------------------
  [ 구현 대상 ]
  - 브랜드 관리 그리드 (조회/수정/삭제)
  - 메뉴 관리 그리드 (조회/추가/수정/삭제)
  - 정산 내역 조회 그리드 (읽기 전용 + 필터)

  [ 공통 패턴 ]
  - 검색 폼 + 그리드 조합
  - 모달 팝업으로 상세 등록/수정
  - 현업 화면 구조와 비교하며 구현

  [ 학습 포인트 ]
  - 재사용 가능한 그리드 컴포넌트 추상화
  - 검색 조건 상태 관리
